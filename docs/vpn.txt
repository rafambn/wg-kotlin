FILE: new-vpn/src/commonMain/kotlin/com/rafambn/kmpvpn/Vpn.kt

REVIEWED
- Public constructor should expose only engine, config, and onEvent - hides interfaceFactory to simplify user API and enforce dependency injection through factory defaults
- state() returns the VPN lifecycle status by checking existence then runtime state - distinguishes between NotCreated, Created (exists but down), and Running (active tunnels)
- exists() on JVM should check if interface actually exists on the OS - ensures accurate lifecycle state reporting across platforms
- isRunning() checks three conditions together - interface must exist on OS, be up at OS level, and have active userspace TUN sessions (BoringTUN) running
- close() function should not delete the interface - prevents data loss on unexpected closes and allows interface reuse, only delete() should remove the interface permanently
- Public constructor should not be changes unless told to
- interface name is restricted to the allowed tun of macOS for all platforms for simpler code - utun[0-9]+
